## 当前可以安全对外表达的成果

- 当前仓库有 66 项结构性与回归测试通过；这些测试证明管道和确定性门禁，不代表生成内容质量或统计准确率。
- macOS 和 Windows Codex CLI 已在文档所列样本上执行插件安装、激活、正向编译和失败关闭式拒绝路径。
- 一次 Windows 修订事务验证了旧状态失效、重新确认、receipt 门禁、版本递增和基础草案不被覆盖。
- 多工作流输入可以进入 `needs_workflow_selection`，在用户选择前禁止编译。
- 一个真实表情包任务完成了 `N=1` 留出测试，并在两轮用户修正后被接受。

## 明确不能对外扩大为以下结论

- 不能称为 `Proven` 或 `Field-tested`。
- 不能声称分类器已经达到统计意义上的准确率。
- 不能声称所有 ChatGPT、Codex 或操作系统环境都已完成端到端验证。
- 不能声称确定性门禁能够发现模型没有声明出来的第二个工作流。
- 不能把一次真实任务的通过描述为跨角色或跨领域泛化成功。

## 提交前 P0 必做项

1. 公开 GitHub 仓库已创建并回填：`https://github.com/tsirou694-prog/flowprint`；MIT License 已补充。
2. 清理仓库里的个人路径、账号信息、认证输出和不必要的原始证据。
3. 从全新目录执行一次安装、Node 8 双工作流选择、拒绝和选中后编译流程。
4. 冻结最终插件版本，使 manifest、README、视频和截图中的版本一致。
5. 按脚本录制并公开上传不超过 3 分钟、带声音的 YouTube 演示。
6. 在主要 Codex 构建线程运行 `/feedback`，保存 Session ID。
7. 把仓库、视频和 Session ID 填入 Devpost 文案。

## 建议的精修顺序

1. **先冻结演示路径**：只保留一个最短、最稳定的评委复现流程。
2. **再精修视频**：先录制无剪辑试跑，确认可控制在 2:45–2:55。
3. **再精修 Devpost 文案**：根据实际录屏内容删去未展示的主张。
4. **再做视觉素材**：封面、架构图和四张关键证据截图。
5. **最后做发布审计**：链接、许可证、隐私、版本和哈希一致性。

## 推荐演示主线

1. 激活 FlowPrint，并展示宽泛个人目录被 scope guard 阻止。
2. 在明确的项目目录里输入岳阳亲子游多成果样本。
3. 展示两个候选工作流以及 `needs_workflow_selection`。
4. 尝试编译未选择输入，展示 `workflow_selection` gate 拒绝且不生成目录。
5. 明确选择“亲子游规划”，编译单一 Skill 草案。
6. 展示 manifest 中 `install_state: not_authorized`，说明编译、安装、发布是三份独立授权。
7. 用一句话展示修订 receipt 和 `N=1` 真实任务结果，避免在三分钟视频里展开全部历史。
