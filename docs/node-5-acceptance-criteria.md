# FlowPrint 节点 5：Skill 编译器验收门

日期：2026-07-17  
状态：进入实现前的验收契约

## 节点目标

把状态为 `ready` 的结构化分类结果编译成可检查的 Skill 草案及其 Profile、运行参数、权限边界和依赖清单。节点 5 不以“生成了一份 SKILL.md”为完成标准，而以“无效输入确实无法越过编译门、生成结果可追踪、修改影响可判定”为完成标准。

## 输入前置条件

编译器只接受同时满足以下条件的 `.flowprint/classification.json`：

1. `schema_version` 为节点 4 契约支持的版本；
2. `gates.decision` 为 `ready`；
3. 所有需要复核的 `item_id` 已有用户确认记录；
4. `validate_classification.py` 在编译调用链中实际执行并以退出码 0 完成；
5. 校验器没有执行、超时、崩溃或返回非零时一律 fail closed，禁止编译。

聊天预览看起来合理不能替代结构化 JSON，也不能替代校验器的真实执行证据。

## 五项强制工程门

### G1 — Failure Lesson 完整性

每个 Failure Lesson 必须包含独立的 `abstracted_rule`。如果聊天预览省略该字段，编译器不得根据显示文本猜测补齐后静默继续；必须回到分类修复或明确生成待确认的修复项。

### G2 — 共享确认的显式映射

一个确认问题可以覆盖多个相关项目，但必须在机器结果中用 `item_ids` 完整记录。任何 `review_required: true` 的项目未被问题或确认记录引用时，编译器拒绝继续。

### G3 — 校验器是流水线硬门

编译器必须通过受控子进程或等价的确定性调用执行分类校验，并保留退出码及错误摘要作为编译记录。禁止只在开发测试中运行校验器、实际编译路径绕过它。

### G4 — 依赖图与影响范围

生成的 manifest 必须表示：

- Skill 依赖哪些 Profile 字段及其版本；
- Core、Domain、Profile、Run Parameter、Failure Lesson、Permission Boundary 的来源项目；
- 每个生成文件由哪些分类项产生；
- 修改分类项后哪些产物变为 `stale`、哪些测试必须重跑。

没有依赖映射时，不得声称支持“自然语言修改后自动级联更新”。

### G5 — 有能力时仍服从权限与确认门

节点 5 已具备写出 Skill 草案的能力，因此必须验证：

- 分类未 `ready` 时不生成草案；
- 校验失败时不产生可安装产物；
- 生成草案不等于安装；
- 安装、覆盖、提交、上传或发布仍需要对应阶段的明确授权；
- 被拒绝后不得通过降级文件名、改写路径或省略字段绕过门禁。

## 必须执行的恶意/坏数据测试

这些测试必须走真实编译入口，而不是只单独调用校验器：

1. 注入数字 `confidence` 字段：编译拒绝，且不留下可安装 Skill。
2. 把一次性偏好以 R1 直接放进 Core：编译拒绝。
3. 删除 Failure Lesson 的 `abstracted_rule`：编译拒绝。
4. 高影响项设为 `review_required: false`：编译拒绝。
5. 共享确认问题漏掉一个 `item_id`：编译拒绝。
6. `needs_review` 伪装成可编译输入：编译拒绝。
7. 设置 `raw_transcript_stored: true`：编译拒绝。

每个失败测试必须同时断言：退出状态非零、错误指向具体规则、产物目录没有可安装结果。

## 高风险语言理解场景

节点 5 结束前至少执行以下五类夹具中的三类；建议全部覆盖：

1. **否定和反讽**：例如“当然不要再把白边当高级质感”，不能把“保留白边”提取为规则。
2. **后续修正覆盖早期要求**：早期要求长尾，最终明确改为无尾；最终指令优先，且冲突/顺序不清时必须复核。
3. **长对话或压缩造成最终决定缺失**：不能自行补全；必须标记证据缺口并停止编译。
4. **单张图片误升格为长期 Profile**：只有视觉观察时必须进入复核，不能直接写成稳定角色事实。
5. **强度词混淆**：“通常”“尽量”“必须”必须保留不同约束强度，不能统一编译成硬规则。

这些夹具用于暴露已知风险，不足以支撑统计准确率结论。报告必须披露样本数量、通过数量和仍未覆盖的场景。

## 节点 5 通过标准

- 有效 `ready` 输入可生成结构完整、规范可校验的 Skill 草案；
- 所有坏数据均在真实编译入口被拒绝，且无可安装残留产物；
- 至少三类高风险语言场景被前向测试并如实记录结果；
- 依赖图能把分类项、Profile 版本、生成文件和测试状态连接起来；
- 自然语言修改能展示影响范围，并对核心或高影响变更强制重新验证；
- 生成、安装和外部执行三种权限状态被明确分离；
- 报告使用“在已测试样本上符合预期”等限定语，不宣称普遍可靠。
